## 1. Comprehensive WebAuthn Implementation

[cite_start]WebAuthn provides a non-transferable authentication method, leveraging a device's Secure Enclave for biometric data like FaceID or Fingerprints[cite: 138, 164, 184].

### A. Database Layer

[cite_start]To store cryptographic credentials securely, you must implement a dedicated migration[cite: 195]. [cite_start]Unlike passwords, these credentials verify physical identity linked to the `User` model[cite: 190, 196].

[cite_start]**Migration: `web_authn_credentials` table** [cite: 197]

- [cite_start]**`id` (UUID)**: The primary key for the specific credential[cite: 200].
- [cite_start]**`user_id` (ForeignID)**: Links the biometric data to a specific employee[cite: 200].
- [cite_start]**`public_key` (Text)**: Stored on the server to verify signatures sent by the device[cite: 182, 201].
- [cite_start]**`credential_id` (String)**: A unique identifier generated by the WebAuthn API[cite: 202].
- [cite_start]**`counter` (Integer)**: Tracks usage to prevent replay attacks[cite: 202].

### B. Implementation Flow

[cite_start]The process involves a cryptographic "handshake" between the Laravel API and the Vue 3 PWA[cite: 175, 180]:

1.  [cite_start]**Challenge Generation**: The Laravel backend creates a unique cryptographic challenge[cite: 180].
2.  [cite_start]**Credential Creation**: The Vue PWA uses `navigator.credentials.create()` to prompt the user for biometrics[cite: 181].
3.  [cite_start]**Verification**: The backend verifies the signed response using the stored public key and issues a secure **Sanctum token** for subsequent API requests[cite: 182, 185].

## 2. Industry-Grade Payroll Service

[cite_start]The `PayrollService` is a dedicated logic layer that keeps your controllers lean by handling complex financial calculations independently[cite: 204].

### A. The Core Calculation Formula

[cite_start]The service applies a standard industry formula to ensure precision[cite: 159, 205]:
$$\text{Net Salary} = (\sum \text{Work Hours} \times \text{Hourly Rate}) + \text{Bonuses} - \text{Deductions}$$

### B. Service Responsibilities

[cite_start]To maintain data integrity, the service performs the following tasks[cite: 194, 206]:

- [cite_start]**Data Aggregation**: It queries the `Attendance` model for the specific month to sum the employee's `total_hours`[cite: 206].
- [cite_start]**Rate Application**: It retrieves the `hourly_rate` (stored in the `User` model) to calculate the base gross salary[cite: 85, 207].
- [cite_start]**Accuracy Adjustments**: The logic automatically identifies late arrivals or unapproved leaves to apply necessary deductions[cite: 94, 208].
- [cite_start]**Record Creation**: The final calculation is stored in the `Payroll` model, tracking the month, year, and payment status (e.g., "paid" or "unpaid")[cite: 88, 209].

### C. System Integration

[cite_start]In your decoupled MVC setup, the components interact seamlessly[cite: 210]:

- [cite_start]**Model**: Provides raw data from `User` and `Attendance` tables[cite: 210].
- [cite_start]**Service**: Processes the data using the financial formula[cite: 211].
- [cite_start]**Controller**: Triggers the service and returns the result as a JSON response to the **Admin Dashboard** in Vue 3[cite: 93, 212].

[cite_start]This infrastructure allows admins to generate reports and calculate payroll for entire departments with a single API call[cite: 157, 158].
